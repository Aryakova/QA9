## Инструменты криптографии

### Кодирование
**Кодирование** - процесс преобразования информации из одной формы представления в другую по определенным правилам с целью обеспечения удобства передачи, хранения или обработки данных. Кодирование не предназначено для сокрытия информации и обычно является обратимым процессом без использования секретных ключей.


### Хеширование
**Хеширование** - процесс преобразования входных данных произвольной длины в фиксированную строку символов (хеш) с помощью математической функции (хеш-функции).


### Шифрование
**Шифрование** - процесс преобразования открытой информации (plaintext) в зашифрованную форму (ciphertext) с использованием алгоритма шифрования и ключа, с целью обеспечения конфиденциальности данных. Обратный процесс называется дешифрованием.

## Симметричное и асимметричное шифрование

### Симметричное шифрование
**Симметричное шифрование** - тип шифрования, при котором для шифрования и дешифрования используется один и тот же секретный ключ. Отправитель и получатель должны заранее договориться об использовании общего ключа.

### Асимметричное шифрование
**Асимметричное шифрование** - тип шифрования, при котором используются пара ключей: открытый ключ (public key) для шифрования и закрытый ключ (private key) для дешифрования. Открытый ключ может свободно распространяться, while закрытый ключ хранится в секрете.

## Плюсы и минусы

### Симметричное шифрование

**Плюсы:**
- Высокая скорость шифрования/дешифрования
- Меньшие вычислительные затраты
- Подходит для шифрования больших объемов данных

**Минусы:**
- Проблема безопасной передачи ключа
- Сложность управления ключами при большом количестве пользователей
- Не обеспечивает цифровую подпись и non-repudiation

### Асимметричное шифрование

**Плюсы:**
- Решает проблему распределения ключей
- Обеспечивает цифровые подписи и аутентификацию
- Поддерживает non-repudiation

**Минусы:**
- Медленнее симметричного шифрования
- Большие вычислительные затраты
- Не подходит для шифрования больших объемов данных

## Алгоритмы симметричного шифрования

### Слабые алгоритмы 

- **DES** (Data Encryption Standard) - 56-битный ключ, уязвим к brute-force атакам
- **RC4** (Rivest Cipher 4) - имеет серьезные уязвимости, легко взламывается
- **3DES** (Triple DES) - устаревший, медленный, с ограниченной безопасностью

### Безопасные алгоритмы 

- **AES** (Advanced Encryption Standard) - 128/192/256-битные ключи, золотой стандарт
- **ChaCha20** - современный, быстрый, безопасный алгоритм

## Алгоритмы хеширования

### Не рекомендованные алгоритмы

- **MD5** - полностью скомпрометирован, уязвим к коллизиям
- **SHA-1** - считается небезопасным, уязвим к атакам

### Рекомендованные алгоритмы

- **SHA-256/SHA-512** (из семейства SHA-2) - широко используются, считаются безопасными
- **SHA-3** - новейший стандарт, высокая стойкость к различным атакам
- **Bcrypt** - специально разработан для хеширования паролей, устойчив к brute-force
